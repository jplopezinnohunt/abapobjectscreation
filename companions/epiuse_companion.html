Who is EPI-USE Labs
About EPI-USE Labs EPI-USE Labs is a global technology company specializing in SAP HCM (Human Capital Management) product extensions. They build certified SAP add-on software delivered entirely as ABAP classes into the SAP standard namespace. Their products are used by large enterprises and international organizations to manage HR data privacy, payroll testing, system landscape optimization, and HCM reporting without custom Z-development.
🏢 Company

South African-origin global SAP partner. SAP-certified solution provider. Specializes exclusively in SAP HCM/Payroll add-ons.

📦 Delivery Model

Transport-based installation. Delivered via their own SAP system (E9BK). Split into 500-object transport chunks due to SAP limits.

⚖️ Licensing

Licensed commercial software. UNESCO pays per-product license. Updates released periodically requiring re-transport into client system.

What Was Installed at UNESCO

Based on transport object analysis: 10,756 CLAS + 465 PROG + 145 ACID + 111 AUTH + 72 TABU + 78 DOCT objects

🔄 Data Sync Manager (DSM) for HCM

Primary product. Allows copying employee/payroll data between SAP systems (PROD → QAS → DEV) while masking sensitive personal data (GDPR compliance). Used to keep test systems current with real data shapes without real data exposure.

HR Data Copying GDPR Compliance Payroll Testing System Refresh

🔒 Data Secure

Data masking and scrambling engine. Works with DSM to anonymize employee names, salaries, bank accounts, and personal identifiers when copying data to non-production systems. Explains the ACID (Authorization Check ID) and AUTH objects in the transports.

Data Masking GDPR Authorization Objects

📊 Query Manager / Variance Monitor

Reporting layer over SAP HCM/Payroll data. Enables non-technical HR staff to run structured queries on payroll results and employee data without needing SE16/SAP transaction access. Explains the DOCT (documentation) and report program objects.

HR Reporting Payroll Variance

⚙️ EPI-USE Framework Classes

Core ABAP class library shared across all EPI-USE products. The 10,756 CLAS objects are this framework — utility classes for UI rendering, RFC handling, data transformation, and SAP Fiori integration. Re-installed with every product update.

Interpretation The dominance of CLAS (88.5%) confirms this is a pure object-oriented ABAP framework. EPI-USE delivers their entire product as reusable class libraries. The ACID + AUTH objects confirm a dedicated authorization concept. TABU entries are product configuration defaults. DEVC packages show they organize code into named SAP packages (visible in SE80). The 26 TRAN entries are the actual user-facing transactions (e.g., the DSM copy cockpit, Data Secure configuration screen).
Strategic Implications for UNESCO
⚠️ Upgrade Dependency Risk EPI-USE delivers updates quarterly/annually. Each upgrade touches 1,000–3,000 objects. If UNESCO's own Z-code extends or calls EPI-USE classes directly, upgrades can break custom code. Dependency map should be created.
⚠️ License continuity The most recent activity is 2025-Q3. If the license expires or EPI-USE deprioritizes SAP ECC (in favor of S/4HANA), UNESCO will be running unsupported HCM tooling. This is an S/4HANA migration risk factor.
✅ GDPR Compliance Asset DSM + Data Secure are actively used for HR data masking between systems. This is a compliance requirement. The 2020 COVID spike shows it was critical during emergency remote operations.
