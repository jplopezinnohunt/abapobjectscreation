#2
Bug Yimiao Guo (RPMR) · Jan 30, 2026 Automatable
Statistical Commitment Carried Forward Without Budget Consumption
A trip financed by regular budget was incorrectly identified as a "Statistical Commitment". It was carried forward but consumed zero budget, impacting reporting. The AO was advised to use the current biennium budget to cover the cost.
Improvement Action
Detect statistical commitments early, before the reconciliation phase, with support from AP and DBS. WRTTP filtering in FMIFIIT distinguishes statistical vs. real commitments.

Infrastructure & Server

3 issues
#3
Infrastructure Jeannette La (DBS) · Jan 14, 2026
Server Memory Saturation During Carry Forward
All institutes running carry forward simultaneously caused SAP to block. The server cannot handle concurrent access to the same large tables with different selection criteria. "SAP is clearly not meant to have such similar huge access to same tables."
Improvement Action
Define a staggered schedule with precise planning so institutes do not run carry forward at the same time.
#4
Infrastructure Jeannette La (DBS) · Jan 14, 2026
P01 Full Backup Not in the Agreed Plan
A full backup of the production system was not included in the agreed carry forward plan, causing delays.
Improvement Action
Always include P01 full backup in the carry forward plan.
#5
Infrastructure DBS · Jan 30, 2026
P01 Not Restarted Before/After Carry Forward
Performance degradation after carry forward because SAP buffers were not cleaned.
Improvement Action
Always restart P01 before and after the carry forward exercise to clean all buffers from CF activities.

Process Issues

2 issues
#6
Process Yimiao Guo (RPMR) · Jan 16, 2026
Voluntary Contribution Testing Started Too Late
Budget carry forward testing on voluntary contributions only started in January 2026. All budget types should be tested together in December.
Improvement Action
Communicate the timeline clearly to BSP so that all budget types are tested together in December.
#7
Process Pablo Lopez (DBS) · Jan 19, 2026 Automatable
Fund Validity Period — No Automated Update Mechanism
A program to extend fund validity periods was planned but did not work. Updates were done manually using ALIOS (batch input). Some institutes faced licence issues delaying the fund extension. A universal mechanism usable by all institutes is needed.
Improvement Action
Build an automated fund validity update mechanism (RFC-based or improved program) that can be used by all institutes without licence dependency. Gold DB has all 64K funds for validation.

Enhancements Proposed

3 enhancements
#8
Enhancement Pablo Lopez (DBS) · Jan 16, 2026 Automatable
Automated "Check 0" Commitment Report
At year-end, every commitment document must net to zero: pending amount + carry forward in period 16 = 0. Currently requires manual data download and comparison. An automated report would replace this and significantly speed up post-CF closing.
Improvement Action
Build an automated "Check 0" validation report per commitment document. Can be implemented as a SQLite query against FMIFIIT + FMIOI in the Gold DB.
#9
Enhancement Pablo Lopez (DBS) · Jan 16, 2026 Automatable
Full Background Execution of Carry Forward
In 2025, commitment simulation in background worked very well. The balance carry forward was still manual. For 2026, the full process should run in background:

#10
Enhancement Pablo Lopez (DBS) · Jan 16, 2026 Automatable
Project Carry Forward Controls
No specific controls for project-related carry forward exist. Learnings from 2025 need to be formalized into a plan.
Improvement Action
Define project-specific CF control plan based on 2025 learnings. Owner: Pablo Lopez + Leandre Francisco. Golden Query (FMIFIIT.OBJNRZ to PRPS.POSID) enables WBS-level commitment tracking.
