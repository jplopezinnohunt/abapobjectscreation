7. Detection Method (link tiers explained)

How the matrix decides each pair's tier. Order of precedence, top-down:

TierTestPool model
HARDsubstr(POSID,1,10) = GEBER AND fund TYPE 101-112Full WBS pool, paired 1:1 with fund
DERIVEDWBS has only 1 distinct fund posting in 2024-2026Full WBS pool
DERIVED-MULTIWBS has 2-10 distinct fundsWBS pool × (this fund's consumption / total consumption)
OUTLIERWBS has > 10 distinct fundsNone (separate file, governance decision pending)

Pool computation

PS pool (per WBS, year=2026)
PS pool = SUM(BPJA WTJHR for OBJNR=this WBS, GJAHR=2026, WRTTP='41')      [budget]
        - SUM(COEP WKGBTR for OBJNR=this WBS, GJAHR=2026, WRTTP IN ('04','11'))  [actuals]
        - SUM(COOI WKGBTR for OBJNR=this WBS, GJAHR=2026)                  [commitments]
FM pool (per fund × bucket)
For each (Fund, FundCenter, CommItem) bucket the WBS posts to:
   FM_bucket_pool = SUM(FMIFIIT FKBTR for FONDS=fund, GJAHR=2026, WRTTP='66')   [revenue]
                  - SUM(FMIFIIT FKBTR for FONDS=fund, GJAHR=2026, WRTTP='54')   [consumption]
FM pool (matrix) = MIN over all buckets the WBS uses for this fund.
   (= the worst-case bucket = the bucket that will block first.)
State decision tree
if PS pool > 0 and FM pool > 0:    ALIGNED
elif PS pool > 0 and FM pool <= 0: FM-EXHAUSTED   <- INC-005638 class
elif FM pool > 0 and PS pool <= 0: PS-EXHAUSTED
else:                                BOTH-EXHAUSTED

FM pool (per Fund × bucket, 2026)
SELECT FONDS, FISTL, FIPEX,
  SUM(CASE WHEN WRTTP='66' THEN _signed(FKBTR) ELSE 0 END) AS revenue,
  SUM(CASE WHEN WRTTP='54' THEN _signed(FKBTR) ELSE 0 END) AS consumption
FROM fmifiit_full
