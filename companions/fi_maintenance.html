⚠ FM-AVC vs PS-AVC — Model Gap (MODEL-GAP-2026-001) FI ABEND DEPENDENCY

FI postings against blocked funds will fail with FMAVC005 / FI 026 before reaching the GL. Cross-references: model gap companion · knowledge/incidents/INC-000005638_ses_block_donor_fund_avc_fipex_deficit.md · Zagentexecution/quality_checks/inc5638_fm_ps_avc_misalignment.py

1. The two-engine asymmetry TIER_1

UNESCO has two AVC engines. FM-AVC enforces per (FUND, FUNDSCTR, CommItem); PS-AVC enforces per WBS. Two granularities, by design. They disagree at biennium close-out on TYPE 101-112 donor funds.

2. Footprint TIER_1

Class-detector run 2026-05-01 (inc5638_fm_ps_avc_misalignment.csv, 6,745 rows): 2,005 buckets at risk UNESCO-wide (488 Tier 1 + 1,517 Tier 2). Growth +4.1%/month.

3. Canonical case — INC-000005638 TIER_1

Fund 196EAR4042, Gabon Bureau, 3 POs blocked at ML81N + F110. FM-AVC TC bucket = −$1,561,560.89; PS pool +$48,898 (silent).

4. What this means for FI Maintenance / Support TIER_2

FI postings (FB60, MIRO, F-53, F110) against any account-assignment carrying a blocked FONDS will abend in AVC_DERIVE_BUDGET_AVC before the FI document is created. Symptom: FMAVC005 ("Annual budget exceeded by ... USD") or its UNESCO-localised wrapper, raised at COBL stage. This looks like an FI error but it is FM/PSM. FI maintenance support should: (a) confirm via SU53 / ST22 that the abend message is FMAVC005 (not FI 026 from cost-element/G-L config); (b) read FMAVCT for the (RFIKRS, RFUND, RFUNDSCTR, RCMMTITEM) tuple; (c) route to PSM domain agent for AVC analysis. FI postings DO NOT have a "skip AVC" toggle — the engine is mandatory for FM-relevant document types.
