# Security and Credentials Governance

A critical aspect of an enterprise SAP Automation Framework is the absolute protection of system credentials. The executing agents (Frontend Playwright, Backend node-rfc, CI/CD abapGit) must *never* hardcode passwords, API tokens, or secrets into the generated `Zagentexecution/tasks/` scripts.

## 1. Zero-Hardcoding Rule
No execution script generated by any Worker Agent may contain a raw password or access token.
If a script requires authentication, it must dynamically inject the credentials from a localized, secure configuration source.

## 2. The Credential Vault (`Zagentexecution/config/`)
The primary mechanism for storing secrets is the `Zagentexecution/config/` directory. 
The Orchestrator Agent must instruct the user to create the following JSON files (or `.env` equivalents) manually. These files are designated as local-only and must never be committed to source control:

### A. SAP Authentication (`sap_auth.json`)
Used by the Frontend (Playwright), Backend (node-rfc), and Native (winax) agents.
```json
{
  "system": "ECC 6.0",
  "client": "100",
  "user": "JP_LOPEZ",
  "password": "YOUR_SAP_PASSWORD"
}
```

### B. Notion Authentication (`notion_auth.json`)
Used by the Orchestrator Agent to boot the Notion MCP Server.
```json
{
  "NOTION_API_KEY": "secret_1234567890",
  "NOTION_PAGE_ID": "abcdef123456"
}
```

### C. Git/abapGit Authentication (`git_auth.json`)
Used by the CI/CD Agent to automate the `ZABAPGIT` transaction pushing code to GitHub.
```json
{
  "username": "jp_lopez",
  "personal_access_token": "ghp_1234567890"
}
```

## 3. Dynamic Injection
When writing a script (e.g., `create_segw.js`), the agent must use native Node.js tools like `fs.readFileSync` or `process.env` to load the variables cleanly:
```javascript
const fs = require('fs');
const auth = JSON.parse(fs.readFileSync('../../config/sap_auth.json', 'utf8'));

// Use auth.user and auth.password for the Playwright login flow
```

*By strictly enforcing this rule, the framework ensures that even if the generated `Zagentexecution/tasks/` code is accidentally shared, the SAP system environments remain perfectly secure.*
