# Deep Security Guardrails for AI SAP Execution

Because this framework grants the AI unprecedented control over SAP Web GUI (Playwright) and SAP Backend (node-rfc), we must establish absolute, unbreakable guardrails to prevent accidental damage to the system, data loss, or system lockups.

Before the Orchestrator dispatches any Worker Agent, it must validate the request against these core security rails.

## 1. Environment Constraints (The 'DEV-Only' Rule)
The AI is strictly prohibited from executing generation, configuration, or structural changes in Quality (QAS) or Production (PRD) environments.
- **Rule:** The Orchestrator must parse the target SAP System ID (SID) and Client from `sap_auth.json`.
- **Enforcement:** If the client is mapped to PRD (e.g., Client 800) or QAS (e.g., Client 400), the Orchestrator must immediately halt and throw an `EnvironmentViolationError`, unless the user provides an explicitly typed, multi-stage manual override code. Default execution is restricted to Development (e.g., Client 100).

## 2. Transaction Blacklisting & Whitelisting
The AI cannot freely roam all 100,000+ SAP transactions. 

### A. The Blacklist (Instant Halt)
The UI Automation Agent and Core System Agent are explicitly forbidden from executing or interacting with:
- **SCC4 / SCCL:** Client administration and copies.
- **SE14:** Database utility (High risk of `DROP TABLE` data loss).
- **SU01 (Mass Deletion):** User deletion without explicit human approval.
- **OBR1:** Resetting transactional data.

### B. The Whitelist (Approved for Autonomous Execution)
- **Development Builder:** SEGW, SE11, SE24, SE80, SE38.
- **Git/CI:** ZABAPGIT.
- **Workflow:** SWDD, SWO1.

## 3. Data Destruction Prevention (The 'Never Drop' Mandate)
- **UI Playwright Rule:** The Frontend Worker Agent must NEVER click a "Delete", "Drop", or "Truncate" icon (often the trash can icon in SAP GUI) without pausing the execution and using `notify_user` to request explicit, manual user confirmation.
- **Backend BAPI Rule:** When executing a BAPI that modifies data (e.g., `BAPI_USER_CHANGE`), the Backend Worker must *always* execute the call with the parameter `TESTRUN = 'X'` (or equivalent) first. It only passes `TESTRUN = ' '` and calls `BAPI_TRANSACTION_COMMIT` after validating the test run results.

## 4. Infinite Loop & Resource Lock Prevention
If automated script elements fail to find a locator in SAP Web GUI, or if a BAPI pagination fails, the AI could theoretically loop endlessly, exhausting SAP Dialog Work Processes (DIA) and locking out human users.
- **Playwright Rail:** Every Playwright script generated by the UI Agent must implement strict `timeout: 15000` limits on `waitForSelector`. If an element is not found within 15 seconds, the script must fail gracefully rather than polling infinitely.
- **Data Loop Rail:** Any generated `while` or `for` loop executing BAPIs via `node-rfc` must have a hardcoded ceiling (e.g., `if (iterations > 1000) break;`).

## 5. The "Panic Button" Architecture
Every `Zagentexecution/tasks/` folder script generated must log its current step to the console synchronously. If the human user notices anomalous behavior in the SAP system, they can immediately `CTRL+C` the terminal running the Node script, instantly killing the Playwright browser or breaking the RFC connection without leaving orphan runaway processes in SAP.

## 6. STMS Transport Management Rules (SE09/SE10)
To prevent the accidental deployment of unverified code across the landscape, strict rules govern Transport Requests.
- **Task Requirement:** The AI may freely create Workbench (WK) or Customizing (CUST) requests and assign new objects to them.
- **The 'No-Release' Rule:** The AI is strictly FORBIDDEN from automatically releasing Transports (via SE09 or `BAPI_CTOBJECT_RELEASE`) unless the human user has provided explicit, written secondary confirmation authorizing the release. 
- **CI/CD Triggers:** The CI/CD agent (abapGit) must push code *before* the transport is released, as releasing it locks the objects and prevents further autonomous correction loops.
