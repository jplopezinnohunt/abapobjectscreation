# abapGit Integration

Standard ABAP development relies strictly on SAP Transport Requests (W-*). However, for a modern, enterprise-grade AI automation framework, all code generated by the agent must also be version-controlled in Git.

This skill dictates how the agent should interact with `abapGit` after successfully generating new SAP objects (like SEGW services or ABAP classes).

## 1. When to Use abapGit
The agent should only execute an `abapGit` synchronization **after** it has fully completed writing/generating the new objects in SAP, and the objects are locked in the user's Workbench Request.

## 2. Execution Method
`abapGit` is typically run via the `ZABAPGIT` transaction. 
Since this is a UI-based operation:
1. The agent should generate a secondary Playwright script (or append to the main one) to launch `ZABAPGIT` in the Web SAP GUI.
2. The agent navigates to the specific repository linked to the development package (e.g., `ZCRP`).
3. The agent clicks "Stage", selects the newly generated objects, and commits them.

## 3. Configuration Requirements
To execute a commit, the agent requires the following data from `Zagentexecution/config/`:
- **Repository URL** (e.g., https://github.com/org/sap_repo.git)
- **Git User/Email**
- **Commit Message**: The agent should automatically generate a highly descriptive commit message summarizing the AI's generation task (e.g., *"AI-Generated: Initial SEGW Data Model for Material and Customer"*).

## 4. Workflow Rule
If the user's project utilizes `abapGit`, pushing the code to the repository is considered the **mandatory final step** of any generation task, establishing a perfect CI/CD bridge between AI generation and source control.
