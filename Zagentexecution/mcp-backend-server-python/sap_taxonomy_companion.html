Layer 1 — Object Nature (What is it?)

PrincipleEvery SAP object transported belongs to one of 6 fundamental natures. This is the highest-level filter — separating code from configuration from data from security from documentation from transport system noise.

🔵 DEVELOPMENT

  • Custom code written by developers
  • Changes system behavior programmatically
  • Requires developer key + system access
  • Covered by code review process

🟠 CONFIGURATION

  • Table entries (TABU) adjusting SAP behavior
  • Done by functional consultants via IMG
  • No programming — parameter-driven
  • Examples: chart of accounts, pay scales, org units

🔴 SECURITY

  • Authorization profiles and roles (PFCG/ACGR)
  • Auth objects and check IDs
  • Done by security administrators
  • Examples: P_IKOUNA's 3,981 role transports

🟢 DATA MODEL

  • Database tables (TABL), domains (DOMA), elements (DTEL)
  • Structural changes — impacts DB schema
  • High risk: affects all programs reading the table
  • Requires transport to QA + prod carefully

🔷 USER INTERFACE

  • Text elements, form texts, screen labels
  • Help documentation (SHI3/SHI6)
  • Translations and localized content
  • Low risk but important for user experience

⬜ TRANSPORT METADATA

  • RELE: Release action records (who/when released)
  • MERG: Merge action records
  • Not real SAP objects — CTS bookkeeping
  • Should be EXCLUDED from change metrics

Layer 2 — Technical ABAP Sub-type (What kind?)

📄 Reports & Programs DEV_REP

  • PROG Executable program (SE38)
  • REPS Program include (source part)
  • REPT Text include (UI texts)
  • REPY CUA status/menu include
  • VARX Program selection variants

🔷 Classes & Interfaces DEV_CLAS

  • CLAS ABAP class (SE24)
  • INTF ABAP interface
  • METH Class method (sub-object)
  • CPUB/CPRO/CPRI Class sections
  • CINS/CINC Class includes

⚡ Function Groups / BAPIs / RFCs DEV_FUGR

  • FUGR Function group (SE37)
  • FUNC Function module
  • FUGA Function group include
  • BAPIs are FUNC with BAPI_ prefix
  • RFCs are FUNC with REMOTE_ENABLED=X

🔄 Workflows DEV_WF

  • SWFP Workflow task
  • SWFT Standard task
  • SWED Event linkage
  • SWFL Workflow template
  • PDWS Process definition

🔌 Enhancements / BADIs DEV_ENH

  • ENHO Enhancement implementation
  • ENHS Enhancement spot
  • ENHD Enhancement definition
  • ENHC Composite enhancement
  • These extend SAP standard without modification

🌐 Fiori / BSP / Web Dynpro DEV_UI

  • WAPP SAPUI5/Fiori app (JS+XML)
  • SBYP BSP page
  • SMIM MIME objects (CSS, icons, JS)
  • WDYC/V/A Web Dynpro component/view/app
  • IWSV/IWOM OData service registration
  • SICF HTTP service node

📝 Forms & Output DEV_FORM

  • SFPF Adobe Interactive Form
  • SSFO Smart Form
  • F30 SAPscript (pay slips, FI forms)
  • DMEE Payment medium format (DMEE)
  • PFRM Print form

⚙️ General Customizing CFG_GEN

  • TABU Table entries — primary config carrier
  • CUS0/1 Customizing objects
  • NROB Number range definitions
  • TDAT Customizing table data
  • LODE BRF+ logical objects
  • SCVI Screen display variants

👥 HCM Configuration CFG_HCM

  • PDTS/PDVS Personnel dev objects
  • TABD HR cluster table data
  • SOTT HCM object type definitions
  • DTED/DOMD HR dictionary objects
  • Pay scales, work schedules, absence types

🔒 Security & Roles CFG_SEC

  • ACGR Activity Groups (= Roles, PFCG)
  • AUTH Authorization objects
  • ACID Authorization check IDs
  • SUSC User class assignments
  • 6,171 objects — primarily P_IKOUNA

🗄️ Databases & Data Model DATA_MDL

  • TABL Transparent database table
  • DOMA Domain (value range definition)
  • DTEL Data element (field description)
  • INDX Database index
  • VIEW Database/maintenance view
  • High risk — structural DB changes
