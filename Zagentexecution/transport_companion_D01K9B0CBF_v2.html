🔴 Discovery #1: GL Accounts Without Commitment Items

The Functional Area was created correctly. However, when the 541 GL company code accounts (SKB1) were created/copied, the commitment items were NOT present. In UNESCO's PSM-FM model, every GL account MUST have a commitment item assigned via derivation rules.

Why this matters: In UNESCO's SAP landscape, Funds Management (PSM-FM) is the backbone of budget control. Every financial posting flows through FM via commitment items. Without commitment items mapped to GL accounts, the entire FM budget control layer is disconnected from FI postings for STEM.

The GL → CI → FM Relationship

How it works: In SAP PSM-FM, the chain is: GL Account (SKA1/SKB1) → Commitment Item (FMCI) → FM Derivation (FMDERIVE). When a user posts to a GL account, SAP's derivation engine (FMDERIVE) looks up which commitment item corresponds to that GL account. The commitment item is the FM equivalent of the GL account — it is the budget line that tracks spending. Without this mapping, FM does not know which budget line to check or consume.
Fix Actions:
  1. Verify scope: Determine if STEM uses shared commitment items from FM Area 01 (other company codes) or needs its own set
  2. Create commitment items: Transaction FMCI — create commitment items for each GL account category (revenue, expense, balance sheet if needed)
  3. Update derivation rules: Transaction FMDERIVE — add rules that map GL accounts to commitment items for company code STEM
  4. Verify: Post a test document to a P&L GL account and confirm the FM commitment item is derived correctly

TCodes: FMCI (create commitment items), FMDERIVE (derivation rules), FMCID (display CI), S_ALR_87013611 (verify FM postings)

🔴 Discovery #2: No Cost Elements Created/Copied

EC01 (company code copy) duplicates customizing tables but does NOT copy cost element master data. Cost elements are master data, not customizing. This means the controlling area STEM exists, cost centers can be created, but no P&L account can post to CO because the corresponding cost elements do not exist.

Why this matters: In SAP, every posting to a P&L account (revenue or expense) that requires a CO assignment (cost center, internal order, WBS element) MUST have a corresponding cost element. Primary cost elements mirror GL accounts. Without them, SAP rejects the posting with a hard error: "Cost element XXXXXXXXXX does not exist in controlling area STEM". This blocks ALL expense and revenue postings that require CO assignment — which in UNESCO's setup is virtually every P&L posting.

Key Insights from All 3 Blueprints

1. Cost Elements exist for ALL institutes — ICTP has 368 (326 primary, 33 revenue, 9 statistical), MGIE has 362 (290 primary, 64 revenue, 8 statistical), ICBA has 357 (292 primary, 64 revenue, 1 statistical). They were created post-import. STEM needs ~350–400 cost elements. Use OKTZ to auto-create from GL accounts.
2. Commitment Items are standardized — UNES has 26, all institutes have 22–25. This is a SHARED pattern across UNESCO. STEM should get ~22 commitment items mirroring the existing structure.
3. UNESCO does NOT use Profit Centers — CEPC = 0 for ALL 9 company codes including UNES. Don't waste time creating them.
4. Fund Centers vary by office size — ICTP=114 (large), MGIE=7 (small), ICBA=8 (small). STEM should plan for 5–15 initially.
5. Vendors are the biggest master data effort — ICTP has 35K vendors, but most are shared UNES vendors extended. STEM needs to extend key vendors to its company code (XK01).
6. Fixed assets are created organically — they accumulate over years (ICTP=3,494 in 13 years, class 00002100 dominant). No need to pre-create. Assets are created via AS01 as they're purchased.
7. Funds table is FMFINT (not FMFINCODE) — FMFINCODE returned 0 for all areas via RFC. The Gold DB “funds” table (sourced from a different extraction) shows: ICTP=2,957, MGIE=269, ICBA=53 funds. STEM will need funds created via FM5I.
8. Payment configuration has 3 tiers:
  • Tier 1 (ICTP-style): Full F110 with 19 payment methods, 29 bank rankings — for offices that process thousands of payments automatically
  • Tier 2 (UNES/IIEP/UIS/UBO-style): Partial F110 with 10–17 methods, 10–15 rankings — for medium offices
  • Tier 3 (MGIE/ICBA-style): No F110, 3 methods (M,O,U), 0 rankings — manual payments only via FBZ2/FB60

STEM is currently Tier 3. The question is: should it stay Tier 3 or upgrade to Tier 2?

9. Each institute has its OWN Controlling Area AND FM Area — they are NOT shared with UNES. STEM correctly creates its own CO area and FM area in the transports.
10. CORRECTION: ALL institutes have their own FM area — confirmed by FM consultant. ICBA's FMIFIIT showed 0 in our Gold DB because our extraction script did not include ICBA in the filter (only UNES, UBO, IBE, ICTP, IIEP, MGIE, UIS were extracted). ICBA has its own FM area with its own commitment items (22), fund centers (8), and funds (53). STEM will follow the same pattern: own FM area, own master data.
