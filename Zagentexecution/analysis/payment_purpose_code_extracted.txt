1 GENERAL DESCRIPTION
1.1 Background
The UNESCO payment process has transactions in many different currencies, banks and payment
file formats.
For the payments in USD by Societe Generale, the more and more beneficiary countries require a
payment purpose code to indicate the origin of the payment. The payment purpose code (PPC)
differs form country by country as well where the tag and it’s content to be used.
When these codes are not part of the payment file, there is a high risk that the payment will be
rejected.
In the current version of SAP there is not a standard solution for this so we will create a custom
solution to respond to the requirements.
1.2 Description of the current situation
N/a
1.3 Description of the target development
The payment purpose code should be indicated in the invoice by the user, who will enter the
invoice. We will use the SCB indicator to for the PPC and with the use of a validation rule, the
user will be required to fill out the PPC for the relevant countries.
The payment program will be executed and the table REGUP will contain the SCB indicator (PPC)
in field LZBKZ. The table REGUP represents each payable document that will be paid out.
In a specific table, to be created, we will indicate how the tag where the PPC should be displayed
per country. Another table will be created where it should be indicated for each beneficiary

As each beneficiary country has different requirements, it should be determined by country

The payments generated from the payroll, will need to have a PPC. It will need to be determined

During the generation of the payment file format /CGI…. It should be updated with the PPC. For

description.

the transfer to the payment file. As some countries will only need the PPC and others the
description, we’ll need a table where we can indicate how the PPC should be build and in which

Solution Proposal
Payment Purpose code setup and entry
• The use of SCB indicator to reflect the payment purpose code (PPC)
• Each code is preceded by the country ISO code and 1 number
• Only a maximum of 10 codes per country possible
• The SCB indicator has to be entered for each invoice
• A rule has been created that the SCB indicator (PPC) is required for the
concerned countries
• The list of PPC is provided by BFM and maintained by DBS in SAP
• Only 1 working list should be managed to avoid double or missing entries
